Introduction

vojs는 뷰-오브젝트 자바스크립트의 약자이며 클라이언트 영역의 UI 구현을 위한 라이브러리입니다. 참고로 vojs는 ExtJS나 YUI와 같이 다양한 UI 컴포넌트를 제공하는 라이브러리가 아닙니다. vojs는 뷰-오브젝트라는 개념을 도입하여 UI 컴포넌트를 보다 쉽게 구현할 수 있는 방법을 제공합니다.

Development Issues

단순한 웹 페이지가 아닌 SPA(Single Page Appliction) 방식의 동적인 웹에 대한 수요가 증가함에 따라 Flex나 Silverlight와 같은 RIA 기술이 부각되었지만, 최근 스마트폰 사용자가 폭발적으로 늘어남에 따라 앞서 말한 RIA 기술은 다소 무겁다라는 이유로 걷어내고 있는 추세입니다.

그렇기 때문에 브라우저에 기본적으로 내장된 자바스크립트와 스타일시트만을 이용한 웹앱을 구현하는 방식이 다시 이슈가 되었지만, 복잡한 UI를 만들기 위해서는 다양한 로직들이 클라이언트 영역에 구현되어 있어야 하기 때문에 개발과 디자인/코딩의 구분이 모호해지는 현상이 발생하게 되었습니다.

더군다나 수많은 HTML 태그를 구분하기 위해 무분별하게 키 값을 지정하고, 이를 셀렉트하여 동적으로 데이터를 변경하는 코드가 난무함으로써 뷰와 로직의 코드가 혼재되어 개발 및 유지보수의 어려움이 따릅니다. vojs는 이러한 뷰와 로직의 구분을 명확하게 해주며, 보다 직관적으로 개발자 관점에서 UI를 구현할 수 있습니다.

Supported Browsers

vojs는 모든 브라우저에서 동작하는 자바스크립트 라이브러리입니다. (IE6++, Chrome, Firefox, Safari, ...)

How to Use

vojs를 사용하기 위해서는 먼저 jQuery와 Underscore.js를 로드해야합니다. Underscore.js는 다양한 유틸리티 함수들을 제공하는 심플한 라이브러리이며, vojs의 기본 템플릿 엔진으로 사용됩니다. 그리고 jQuery는 워낙 많이 사용하는 라이브러리이기 때문에 별도로 설명을 하지 않겠습니다.

Concept

뷰-오브젝트는 웹 페이지에서 뷰가 되는 HTML 태그와 뷰를 구성하는 데이터, 즉 '뷰와 객체는 동일하다'라는 것이 기본 컨셉입니다. 먼저 아래 HTML 코드를 보도록 하겠습니다.

					<tr id="person1">
						<td data-bind="name"></td>
						<td data-bind="age"></td>
						<td></td>
					</tr>
					<tr id="person2">
						<td data-bind="name"></td>
						<td data-bind="age"></td>
						<td></td>
					</tr>
위의 HTML 코드는 person1과 person2의 이름과 나이를 보여주는 뷰입니다. 그리고 HTML 태그와 매칭할 객체의 클래스를 정의하도록 하겠습니다.
					var Person = function(id, name, age) {
						vo.applyTo(this, id);
						  
						var self = this;
						  
						self.name = name;
						self.age = age;
						  
						self.bind.name(self.name);
						self.bind.age(self.age);
						  
						self.act.addAge = function() {
							self.bind.age(++self.age);
						}
					};
Person이라는 클래스를 만들었으니 HTML 태그에서 ID 값이 person1, person2인 뷰의 객체를 생성하겠습니다. 
					var person1 = new Person("person1", "James", 21),
	    				person2 = new Person("person2", "Thomas", 15);
James와 Thomas라는 이름을 가진 객체를 생성하였다면, 아래와 같은 예제가 실행됩니다. (Link)
Add 1-year 버튼을 클릭하면 대상의 나이가 증가합니다. 먼저 Person클래스의 2라인에서 호출된 vo.applyTo(this, id); 코드는 객체가 생성이 될 때, 해당 객체의 뷰를 전달 받은 ID 값의 HTML 태그로 설정합니다. 뷰가 설정이 되면 객체는 동적으로 뷰를 변경할 수 있습니다.

뷰의 데이터 속성을 data-bind로 설정을 하면, 해당 뷰를 포함하고 있는 객체는 bind.name 메소드를 가지게 됩니다. 그리고 self.bind.name(self.name);과 같이 호출하면 뷰의 내용이 변경됩니다.

참고로 이러한 규칙은 bind 외 다른 요소들도 동일하게 적용됩니다.

Component

vojs는 Action, Binding, Template, Tag 이렇게 총 4가지로 구성되어 있습니다. 뷰-오브젝트를 적용하기 위해서는 먼저 HTML 태그에 각각의 구성 요소에 맞는 데이터 속성을 설정해야 합니다.

View Setting

HTML 태그는 데이터 속성을 가질 수 있는데, 해당 속성의 기본적인 규칙은 다음과 같습니다.

					Value
HTML 태그가 가지는 데이터 속성 종류는 아래 표와 같습니다.
Name Attribute
Action data-act
Binding data-bind
Template data-tpl
Tag data-tag
key는 Template을 제외한 모든 요소들이 선택적으로 가질 수 있으며, type은 Action과 Binding에만 존재합니다. 그리고 name은 모든 요소가 필수적으로 가지고 있어야 합니다.

Object Setting

뷰-오브젝트를 적용하는 vo.applyTo(object, [element id]) 메소드의 element id를 지정하지 않으면 기본적으로 body 태그를 해당 객체의 뷰로 설정합니다.

객체에 뷰-오브젝트를 적용하는 방법은 다양합니다. 먼저 아래는 일반적으로 사용하는 클래스 내부에 설정하는 방법이며 뷰-오브젝트를 적용할 객체가 여러 개일 경우에 사용합니다. 

					<body>
						

						

					<body/>
				
					var Body = function(id) {
						vo.applyTo(this, id);
					}
					
					var b1 = new Body("body1"),
						b2 = new Body("body2");
두번째로 객체리터럴에 적용하는 방법입니다. 이와 같은 방법은 비교적 단순한 로직을 가지는 단일 UI에 적용할 때 사용합니다. 
					<body>
						
					</body>
				
					var body = {};
					vo.applyTo(body);

Tag

태그는 엘리먼트를 고유 값으로 설정할 수 있습니다. 그냥 id를 지정하여 셀렉트 하는 방법도 있지만 해당 엘리먼트가 보다 명확하게 UI와 관련된 뷰임을 명시해줄 수 있습니다. 또한 연관성 있는 엘리먼트들을 동일한 name의 뷰 속성에 key로 구분할 수 있기 때문에 좀 더 구조적인 UI 구현이 가능합니다.
뷰-오브젝트의 tag 프로퍼티를 통해 id 또는 data-tag 속성이 설정된 HTML 코드를 가져올 수 있습니다. 
					var Body = function() {
						vo.applyTo(this);
						
						var name = this.tag.name.value,
							age = this.tag.age.value,
							location = this.tag.location.value;
							
						alert(name + ", " + age + ", " + location);
					}
					
					new Body();

Advanced

만약에 뷰 속성에 key가 있다면 해당 태그는 단일 객체가 아니고, key가 존재하는 배열 형태의 객체가 됩니다.
이렇게 key를 지정함으로써 info이라는 객체의 맴버들로 해당 폼 태그를 좀 더 구조적으로 가져올 수 있습니다. 
					var person = obj.tag.info;
					
					alert(
						person["name"].value + ", " + 
						person["age"].value + ", " + 
						person["location"].value
					);

Action

액션은 자바스크립트 이벤트와 비슷한 개념입니다. 뷰 데이터 속성에 값을 입력하고, 해당 엘리먼트에 이벤트를 발생시키면 뷰-오브젝트가 적용된 객체에 해당 name과 동일한 메소드가 호출됩니다. 

					Start
Start 링크를 클릭하면, Body 클래스에 정의된 start메소드가 호출됩니다.

						this.act.start = function(e) {

Template

vojs는 Underscore.js 기반의 템플릿 엔진을 지원합니다. 템플릿은 테이블이나 리스트 형태의 데이터를 출력하는 UI에 사용하면 매우 편리합니다. 템플릿을 사용하기 위해서는 body 태그 안에 템플릿 코드가 포함되어 있는 <script data-tpl='viewId:tplName' type='text/template'></script>를 정의하면 됩니다.

viewId를 설정하지 않으면 기본적으로 해당 템플릿이 속한 뷰는 body가 됩니다. 그럼, 먼저 body라는 ID 값을 가지는 뷰에서 사용할 수 있는 personList 템플릿을 정의합니다. 

					<script data-tpl="body:personList" type="text/template">
						<% for(var i in persons) { %>
						<tr>
							<%= persons[i].name %>
							<%= persons[i].age %>
							<%= persons[i].location %>
						</tr>
						<% } %>
					</script>
그리고 템플릿과 템플릿을 적용할 태그의 데이터 속성을 똑같이 설정합니다.

					var Body = function() {

Other features

Search

템플릿을 제외한 나머지 구성 요소들은 기본적으로 get과 val 메소드를 가지고 있습니다. 두 메소드는 해당 데이터 속성을 지정한 뷰를 검색하여 가져올 수 있는 기능을 제공합니다. 차이가 있다면 get은 엘리먼트 객체를 가져오고, val은 해당 엘리먼트의 내용(innerHTML 또는 value)을 가져옵니다. 

					

						
						
						
						
						Submit
아래 코드에서 elem 변수는 submit의 이름을 가진 태그 객체이며, data 변수는 해당 태그의 내용인 문자열 Submit을 가지게 됩니다. 
					var elem = obj.act.get("submit"),
						data = obj.act.val("submit");

						elemList["age"].value + ", " + 
						elemList["location"].value
					);
					
					alert(

Include File

HTML 코드가 너무 많아서 이를 파일로 나누어 관리를 하고 싶을 때, 해당 파일들을 로드하여 현재의 문서에 포함시켜주는 기능을 제공하는 메소드입니다. 

					vo.includePath = "view";
					vo.includeExt = "tpl";
										
					vo.includeTpl("fileName1", "fileName2");
				
					vo.includeTpl("fileName1", "fileName2", { path: "view", ext: "tpl" });
위의 두 코드는 동일한 기능을 수행하는데, fileName1과 fileName2 파일의 내용을 동적으로 body 태그 안에 포함시킵니다. 먼저 옵션에 대한 설명을 하자면 path는 로드 파일이 존재하는 폴더의 경로이며, tpl은 로드 파일의 확장자입니다. (Link)

그리고 body가 아닌 임의의 id를 가진 태그에 파일을 로드하는 기능도 제공합니다. 아래는 test1과 test2라는 id를 가진 태그에 해당 파일을 로드하는 코드입니다. (Link) 
					vo.includeTpl("test1:fileName1", "test2:fileName2");

View-Template

먼저 뷰-오브젝트가 추가될 태그를 body 안에 생성합니다.
